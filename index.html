About

lan.js is simple a Javascript file which runs under node.js which runs a local server that's accessible through private network and loopback IPs. A directory is specified to the application, and it will serves the files in the directory over HTTP.

It's also useful for testing webpages which may be required to not be a file: URI to work.

Required software

  • node.js – runs the server application; get a binary executable for your operating system

Downloads

  • Package – zip file containing node.exe, lan.js, run.bat, and a files folder; prepackaged setup for Windows
  • lan.js – the main server script which runs under node.js
  • run.bat – a simple execution batch script which runs the server when double clicked

Usage

If you're using Windows, select the “Package” download above and unzip it to a folder. You can place files inside the included “files” folder and they will be easily served.
To start the server: double-click run.bat
To close: close the command window, or CTRL+C inside of it if you like

To access the server while it is running, go to one of the following urls: assuming the server is using port 80

  • http://192.168.1.?/ – an IPv4 local network address sometimes looks like this
  • http://[fe80::????]/ – an IPv6 local network address sometimes looks like this
  • http://localhost/ – the generic loopback address (same-computer access only)
  • http://127.0.0.1/ – using the IPv4 loopback (same-computer access only)
  • http://[::1]/ – using the IPv6 loopback (same-computer access only)

Network addresses will be listed on lan.js startup. Pick any address not labeled “external” to use for connecting.

If you're using another operating system, reading the contents of run.bat and porting it to a different shell language should be fairly simple.

Config

The following are the basic editable settings that appear in run.bat. They can be changed if desired or required.

  • NODE
    Typically it will just point to “node.exe”; if you have node.js installed somewhere else, it may require changing
  • PORT
    The port to run the server on. Default is 80, and the valid range is [0,65535] inclusive.
    If an application already is using the specified port, you may get an error such as EADDRINUSE.
    For example, a setting in Skype can cause this to occur (“Options” -> “Advanced” -> “Connections” -> “Use port 80 and ...”)
  • SHARE_PATH
    The path to the directory which should be served. If the directory is shallower than the directory the lan.js script appears in, an error is issued. To ignore this error, the --allow-parent-directories flag must appear on the command line.
